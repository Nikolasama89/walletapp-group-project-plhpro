# Wallet App (Python) — Ομαδικό Project (ΕΑΠ / ΠΛΗΠΡΟ)

## Περιγραφή
Το παρόν repository αφορά ομαδικό project για το **Ελληνικό Ανοικτό Πανεπιστήμιο (ΕΑΠ)**, στην ενότητα **ΠΛΗΠΡΟ**.  
Στόχος είναι η ανάπτυξη εφαρμογής τύπου **Wallet App** σε **Python**, με χρήση της βιβλιοθήκης **Τkinter** για γραφικό περιβάλλον(GUI) και αποθήκευση δεδομένων σε **SQLite**.

Η εφαρμογή θα υποστηρίζει διαχείριση οικονομικών δεδομένων (έσοδα/έξοδα), κατηγοριών, υποχρεώσεων/στόχων και παραγωγή αναφορών/στατιστικών (συμπεριλαμβανομένων γραφημάτων).

---

## Στόχοι & λειτουργικότητα
### Βασικές λειτουργίες
- Καταχώρηση και προβολή **εσόδων/εξόδων**
- Διαχείριση **κατηγοριών**
- Υποχρεώσεις / επιθυμίες (όπου απαιτείται από την εργασία)
- Αναφορές (σύνολα, ανά κατηγορία, ανά χρονική περίοδο)
- GUI για εύκολη χρήση

### Features για υψηλότερη βαθμολογία (στόχοι)
Για υψηλή βαθμολογία, στοχεύουμε να υλοποιήσουμε τα παρακάτω:
1. **Πλήρες CRUD για κατηγορίες**
2. **2 γραφήματα**
   - Pie chart: κατανομή ανά κατηγορία
   - Σύγκριση μηνών: έσοδα/έξοδα/ισοζύγιο
3. **Ειδοποιήσεις για υποχρεώσεις**
   - π.χ. όταν πλησιάζει ημερομηνία λήξης
4. **Budget ανά κατηγορία με προειδοποίηση**
   - προειδοποίηση όταν προσεγγίζεται ή ξεπερνιέται το όριο
5. **Καθαρή δομή βάσης δεδομένων**
   - σωστό schema, σχέσεις/constraints όπου χρειάζεται
6. **Αναφορά με screenshots**
7. **Δοκιμές / test cases** (αν ζητηθούν ή εφόσον υπάρχει χρόνος)

## Κατανομή εργασιών (4 μέλη)
### Μέλος 1 — Βάση Δεδομένων & Αποθήκευση (SQLite)
- Δημιουργία **SQLite schema**
- CRUD συναρτήσεις για όλους τους πίνακες
- Διαχείριση σχέσεων (π.χ. **κατηγορίες ↔ συναλλαγές**)
- Έλεγχος σωστής αποθήκευσης & βασικό validation

### Μέλος 2 — Λογική & Υπολογισμοί (Business Logic)
- Υπολογισμός υπολοίπου (balance)
- Κατανομή ανά κατηγορία
- Φιλτράρισμα ανά μήνα
- Σύγκριση μηνών
- Λογική ειδοποιήσεων
- Budget ανά κατηγορία + προειδοποιήσεις

### Μέλος 3 — Γραφικό Περιβάλλον (GUI)
- Δημιουργία tabs / οθονών
- Φόρμες εισαγωγής δεδομένων
- Σύνδεση κουμπιών με services
- Εμφάνιση γραφημάτων

### Μέλος 4 — Τεκμηρίωση & Testing
- Συγγραφή αναφοράς (documentation)
- Δημιουργία screenshots
- Έλεγχος λαθών / σενάρια χρήσης
- Δοκιμές (test cases)
- Προετοιμασία παρουσίασης (αν ζητηθεί)
